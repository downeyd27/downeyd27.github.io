What is the difference between Margin, Border, and Padding?

When beginning to think about html elements and how they take up space within a web browser, you must first think about the "box model." Every element first contains the content. This is usually the text or image. Right outside the content of an element is the padding, which is the empty spacing leading up to the border. A border can have many attributes. Some including being dashed, dotted, or solid of any thickness and the border can also be any color. After the border is the margin, which is empty spacing between neighboring elements.

What are the best practices associated with using classes vs. ids?

  • Classes: allow you too select multiple elements when using CSS from your HTML files as long as they all have the same class name
  • Ids: allow you to select one unique element with an id name when using CSS from your HTML file. It is possible to use classes and ids on the same elements.

    • I believe through my research a stackoverflow quote put it best, "The rule of thumb is that you should ask yourself: 'is there more than one element which requires the same style, now or at any time in the future?', and if the answer is even 'maybe', then make it a class."

What are the differences between relative, absolute, and fixed positioning?

  • Relative: an element with a relative position can be moved "relative" to its normal position, which elements default at a static position (the order in which the elements appear on the document)
  • Absolute: an element with an absolute position is positioned based off of its first positioned non-static ancestor element
  • Fixed: an element with a fixed position is positioned relative to the browser window and takes the element out of the normal flow of positioned elements. It is as if the other elements ignore a fixed position element.

What does it mean to display inline vs inline blocks?

  • Inline: When displaying an element inline you are allowing elements to sit the the left and right of the element
  • Inline-blocks: When displaying an element inline block you are telling the element to take up the whole row in the browser. The element before will display on the line above and the element after will display on the line below the element.
